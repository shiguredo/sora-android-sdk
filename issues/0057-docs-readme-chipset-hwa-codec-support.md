# README にチップセットごとのコーデック HWA 対応状況を調査して追記する

## 目的

README の「特徴」セクションにある映像コーデックのハードウェアアクセラレーション（HWA）対応状況の記述について、チップセットごとに対応状況が異なる点を調査し、追記する。

これは映像コーデックの HWA 対応状況をまとめた変更の積み残し作業であり、利用者がチップセットごとの HWA 対応可否を把握できるようにすることを目的とする。

## 優先度根拠

- ドキュメントの追記であり、SDK 本体の機能や不具合には影響しないため緊急性は低い。
- チップセットごとの HWA 対応状況の調査には実機やベンダー情報の確認が必要で工数がかかる一方、なくても利用に支障はないため Low とする。

## 現状

- README の「特徴」セクションでは、映像コーデックの HWA 対応状況を以下のように記載している。
  - `README.md`
    - 映像コーデック `VP8` / `VP9` / `AV1` / `H.264` / `H.265` に対応
    - `H.264` と `H.265` はハードウェアデコーダー／エンコーダーに対応
    - `VP9` と `AV1` は対応端末であればハードウェアデコーダー／エンコーダーを利用可能
- 現状の記述は「対応端末であれば利用可能」という粒度に留まっており、どのチップセットでどのコーデックが HWA に対応しているかという具体的な情報がない。
- チップセットごとにコーデックの HWA 対応状況は異なるため、利用者が自分の端末で HWA が効くかどうかを README から判断できない。

## 設計方針

- 主要なチップセットごとに、各映像コーデック（`VP8` / `VP9` / `AV1` / `H.264` / `H.265`）のハードウェアデコーダー／エンコーダー対応状況を調査する。
- 調査結果をもとに、README にチップセットごとの HWA 対応状況を表などの形で追記する。
- 対応状況はチップセットの世代やベンダーの実装によって変化しうるため、調査時点を明記し、網羅性よりも代表的なチップセットを優先して記載する方針とする。

## 完了条件

- 主要なチップセットごとの映像コーデック HWA 対応状況が調査されていること。
- 調査結果が README に追記され、利用者がチップセットごとの HWA 対応状況を把握できること。
