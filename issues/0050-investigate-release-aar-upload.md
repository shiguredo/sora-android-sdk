# リリースサイトに .aar ファイル等をアップロードするか検討する

- Priority: Low
- Created: 2026-06-03
- Completed:
- Polished: 2026-06-03
- Model: Opus 4.8
- Branch:

## 目的

ビルド済みバイナリ（`.aar` 等）を GitHub Release のリリースページにアップロードして配布するかどうかを検討する。

## 優先度根拠

- 現状でも JitPack 経由でビルド済みバイナリを取得できており、配布手段が存在しないわけではないため緊急性は低い。
- 利便性向上を目的とした検討であり、議論段階であるため Low とする。

## 現状

現在はビルド済みバイナリの取得経路が JitPack でビルドしたバイナリを Gradle で取得するルートのみとなっている。リリースページにはビルド済みバイナリをアップロードしていない。

## 設計方針

- JitPack を利用せず、直接バイナリを組み込みたい利用者向けにリリースバイナリを用意することで利便性が向上するかどうかを検討する。
- 他社の Android SDK の配布方法も参考にしながら、リリースページへのバイナリアップロードの是非を判断する。
- JitPack でビルドバイナリを提供している中で、GitHub Release でも提供する利点を整理する。

## 完了条件

- リリースページに `.aar` 等をアップロードするかどうかの方針が決定していること。
- アップロードする場合は、対象とするファイルと配布方法が整理されていること。
