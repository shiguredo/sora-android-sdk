# クイックスタートを自力で作成できるチュートリアルを作成する

- Priority: Low
- Created: 2026-06-03
- Completed:
- Polished: 2026-06-03
- Model: Opus 4.8
- Branch:

## 目的

Sora Android SDK のクイックスタートアプリを利用者が自力で作成できる手順を、チュートリアル形式でドキュメント化する。

既存のクイックスタートは完成済みのサンプルとして提供されているが、ゼロから組み立てる手順がまとまっていない。SDK の導入から接続・配信・受信までを段階的に解説するチュートリアルを用意し、初めて利用する開発者がクイックスタート相当のアプリを自力で構築できるようにする。

## 優先度根拠

- 新規利用者の学習コストを下げる効果が見込まれる。
- 一方で既存のクイックスタートのサンプルコード自体は存在しており、緊急性は高くないため Low とする。

## 現状

- クイックスタート相当のサンプルアプリは存在するが、それをゼロから作成するためのチュートリアルは整備されていない。
- 既存のドキュメントにチュートリアルの参考になる記述が一部あり、それらを下敷きにできる。

## 設計方針

- チュートリアル形式で、空のプロジェクトから順を追ってクイックスタート相当のアプリを構築する手順を解説する。
- 想定する構成は以下。
  1. プロジェクトの準備と SDK の依存追加。
  2. 必要なパーミッション（カメラ・マイクなど）と `AndroidManifest.xml` の設定。
  3. `SoraMediaOption` / `SoraMediaChannel` を用いた接続処理の実装。
  4. 映像・音声の送受信と映像の描画。
  5. 切断処理とライフサイクルの扱い。
- 画面回転による切断の回避（`configChanges` の設定）など、既存の関連 issue でドキュメント化する内容と整合を取る。
- ドキュメントの配置先（`docs/` 配下か別リポジトリのドキュメントか）は実装時に判断する。

## 完了条件

- 空のプロジェクトからクイックスタート相当のアプリを自力で作成できるチュートリアルがドキュメント化されていること。
- 手順どおりに進めれば Sora への接続と映像・音声の送受信ができる状態になること。
