# カメラ以外の入力ソースに対応するか検討する

- Priority: Low
- Created: 2026-06-03
- Completed:
- Model: Opus 4.8
- Branch:

## 目的

カメラ以外の入力ソース（画面共有など）を映像の入力ソースとして利用できるようにするかどうかを検討する。

## 優先度根拠

- 既存機能の不具合ではなく、新規機能の追加検討であるため緊急性は低い。
- 議論段階であり、対応するかどうか自体が未確定のため Low とする。

## 現状

現状の Android SDK の映像入力ソースはカメラを前提とした構成となっており、画面共有などカメラ以外の入力ソースには対応していない。

いくつかの他社 Android SDK では画面共有機能が提供されている。映像入力として、ファイル、カメラ、スクリーン、ビットマップ形式のストリームなど複数の形式に対応している SDK も存在する。

## 設計方針

- 他社の Android SDK が提供している画面共有や、その他の入力ソースの対応状況を調査する。
- Android の画面共有を実現する仕組み（MediaProjection 等）を踏まえ、Sora Android SDK にどのような形で入力ソースを差し替える API を提供できるかを検討する。
- カメラ以外の入力ソースに対応する場合の API 設計と、既存のカメラ入力との両立方法を整理する。

## 完了条件

- カメラ以外の入力ソースに対応するかどうかの方針が決定していること。
- 対応する場合は、対象とする入力ソースの種類と API 方針が整理されていること。
