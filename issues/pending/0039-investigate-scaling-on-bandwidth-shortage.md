## 目的

rotation=0 固定で動かす場合に、帯域不足の状態になったときに映像がスケーリング（解像度の degrade）されるかどうかを調査する。

期待する挙動は「解像度が落ちた映像がエンコードされ配信される」ことであり、これが native buffer の場合でも成立するかを確認したい。

## 優先度根拠

- 帯域不足時の挙動は配信品質に関わるが、現時点では具体的な不具合として報告されているわけではない。
- 検証観点の整理が前提となる調査タスクであり、緊急性は低いため Low とする。

## 現状

- libwebrtc の基本方針として、frame buffer と encoder がともに native であればスケーリングはエンコーダに任せる、と理解している。
- サイマルキャストではない場合に帯域不足で発生するスケーリングについて、native buffer が上がってきたときの挙動が未確認である。
- 期待する挙動の流れは次のとおり。
  - 帯域が不足する。
  - クライアント側の BWE（band width estimate）が絞りにかかる。
  - 通常であれば解像度の downgrade がキックされる。
  - エンコーダの設定が変わる（もしくは再生成される）。

## 設計方針

- 検証に入る前に、確認すべき観点を以下の懸念点に沿って整理する。
  - 本当にスケーリングされるか。
  - スケーリング処理がどこで入るのか。
  - 性能的に問題がないか。
- libwebrtc の該当処理（エンコーダ側でのスケーリング判断）を読み解き、native buffer 時の挙動を確認する。
- 実機で帯域を絞った状態を再現し、解像度が degrade されることを確認する。

## 完了条件

- 帯域不足時に native buffer の映像がスケーリングされるか否かの結論を出すこと。
- スケーリングが入る箇所と、性能上の懸念の有無を整理すること。
