# サンプル集を SDK リポジトリへ統合するか検討する

- Priority: Low
- Created: 2026-06-03
- Completed:
- Model: Opus 4.8
- Branch:

## 目的

別リポジトリで管理している Sora Android SDK のサンプル集を、SDK 本体リポジトリへ統合するかどうかを検討し、方針を結論づける。

## 優先度根拠

- リポジトリ構成の整理に関する方針検討であり、機能やバグに直接影響しないため緊急性は低い。
- ただしサンプルと SDK 本体のバージョン追従コストや、開発体験の改善に寄与し得るため、時間があるときに検討する価値はある。Low とする。

## 現状

- サンプル集は SDK 本体とは別のリポジトリで管理されている。
- 統合の可否を判断するにあたり、以下を満たせる構成かどうかが論点となる。
  - サンプル集から、同一リポジトリ内の SDK モジュールをローカル参照（プロジェクト参照）でビルドできること。
  - サンプル集から、リリース済みの SDK（JitPack に公開されたバージョン）を参照してビルドできること。
- 上記 2 つの参照モードを切り替え可能にする必要がある（開発時はローカル参照、リリース確認時は公開バージョン参照）。

## 設計方針

- 統合した場合のメリット・デメリットを整理する。
  - メリット候補: SDK の変更とサンプルの変更を同一 PR で扱える、バージョン追従の手間が減る。
  - デメリット候補: リポジトリが肥大化する、CI のビルド対象が増える、公開リリース参照とローカル参照の切り替えが煩雑になる。
- Gradle のマルチプロジェクト構成、もしくは設定による参照先切り替え（ローカルプロジェクト参照と JitPack 参照）の実現可否を検証する。
- 統合する場合の移行手順（ディレクトリ構成、CI 設定、ドキュメント）を整理する。

## 完了条件

- サンプル集を SDK 本体リポジトリへ統合するかどうかの方針を結論づけること。
- 統合する場合は、ローカル参照と公開バージョン参照の両方が可能な構成案を示すこと。
